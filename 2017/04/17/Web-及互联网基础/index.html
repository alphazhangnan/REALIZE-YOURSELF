TCP/IP

  • 分层概念
    • 分层好处：某处设计需要改变，只改变该层的设计，不用去关心其他层的变化 – 独立性
    • 应用层 - 向用户提供应用服务时的通信的活动
      • FTP
      • DNS
      • HTTP
    • 传输层 - 对应上面，提供两台计算机之间的数据传输
      TCP - 传输控制协议
      UDP - 用户数据报协议
    • 网络层 - 处理网络上流动的数据包，规定其通过怎样的路径到达目的计算机，并传送数据给用户
    • 数据链路层 - 处理链接网络的硬件部分
  • 传输流：
    • HTTP -> TCP -> IP -> 网络 = 客户端发送请求
    • 网络 -> IP -> TCP -> HTTP = 服务端返回请求内容
    • 请求的发送
      • 应用层HTTP发送请求报文
      • 为传输方便在传输层 TCP 对HTTP报文进行分割，并标明序号及端口，转发给网络层
      • 网络层添加通信目的地的 MAC地址 转发给链路层
      • 服务器在链路断上收到请求数据向上一直发送，
      • 同时去除每个层添加的该层专属报文，当从传输层到达应用层时，服务端正式接收到请求

        HTTP 关系密切的协议:IP、TCP 和 DNS

  • 负责传输的IP协议
    • Base: 网络层
    • Duty: 确保将数据送达对方，需满足一下两个条件
      • IP 地址：节点被分配到的地址
      • MAC 地址：网卡所属的固 定地址
      • ARP: 一种用以解析地址的协议，根据通信方的 IP 地址就可以反查出对应的 MAC 地址
        • 一份数据是不可能一步到位的传送到目的地
        • 中间可能经过多个路由器等网络设施
  • 确保可靠性的 TCP 协议
    • Base: 传输层
    • Duty: 字节流服务，将HTTP报文分段，并确认数据是否送达
    • TCP 三次握手
      • 敬礼：发送 SYN 标志数据包给对方
      • 回礼：返回 SYN/ACK 标志数据包确认收到信息
      • 再见：回传 ACK 标志数据包 握手结束
  • 负责域名解析的 DNS 服务
    • Base: 应用层
    • Duty: 域名到IP地址的解析服务
    • DNS 协议提供通过域 名查找 IP 地址，或逆向从 IP 地址反查域名的服务。

      各种协议与 HTTP 协议的关系

      其实就是输入一个网址，浏览器都作了什么
  • 输入网址，DNS 服务解析域名为IP地址
  • 发送HTTP请求，生成请求的报文，创建TCP连接
  • TCP 将HTTP报文按序号拆分，标记TCP层的专属报头，并确认传送到指定服务器
  • IP 标记专属IP的报头，根据地址一遍搜索服务器位置，一遍中转直到找到，找到后
  • TCP 按序重组报文，进入HTTP范畴
  • HTTP请求相应位置上的资源，并返回

    URI & URL

    URI: 统一资源标识符
  • 字符串表示互联网上某一资源
  • 格式： http://user:pass@www.example.jp:80/dir/index.html?uid=1#ch1
    URL: 统一资源定位符
  • 字符串表示该资源的地点
